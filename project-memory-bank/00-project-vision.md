# 00 — Project Vision

Source of truth: `SYSTEM_PROMPT.md` §1-6 (`System_Prompt/Part1.md`). This file summarizes; it does not replace the constitution.

## What PEE is

An AI-native execution operating system — not a project-management tool. It transforms goals into executable plans, coordinates execution across projects and tools, maintains long-term context, measures progress, and improves future execution. It must scale from a single individual to enterprise teams without architectural redesign (§2).

Positioning (§3): traditional software stores information and remembers tasks; PEE understands work and remembers intent. It is the execution layer for AI-native professionals.

## The execution loop every subsystem must serve (§4)

```
Goal → Planning → Prioritization → Execution → Measurement → Learning → Continuous Improvement
```

If a proposed feature doesn't serve this loop, it doesn't belong in the product.

## Definition of success (§6)

An engineer can install the system, use it daily, and trust it to coordinate meaningful work without losing context. Measured by: reliability, correctness, maintainability, extensibility, developer experience, user experience, performance, observability, security.

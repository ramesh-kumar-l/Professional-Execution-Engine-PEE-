## Architecture

**Resolved 2026-07-17 (`adr/0002`-`adr/0006`); Phase 1 implemented same day, Phases 2-7 implemented 2026-07-18, Phase 8 (Desktop) implemented 2026-07-18, Phase 9 (Mobile) implemented 2026-07-18, Phase 10 (Enterprise) implemented 2026-07-18.** Backend: TypeScript/NestJS modular monolith. Storage: PostgreSQL + SQLite local (`packages/local-client`, consumed unmodified by `apps/desktop`, ported to `expo-sqlite` for `apps/mobile`). Infra: Docker/docker-compose, GitHub Actions, K8s/Terraform deferred. Auth: first-party NestJS module + Auth.js, JWT sessions — Phase 10 added multi-tenancy (`Organization`/`Membership`, request-scoped not session-scoped) and SSO (OIDC via Auth.js's native provider; SAML via a self-built SP behind an OAuth2 façade, since Auth.js has no native SAML provider type). AI: first-party provider interface, Claude + OpenAI. Analytics: plain query/reporting layer, stays owner-scoped in Phase 10 (a documented boundary, not an oversight). Desktop shell: `adr/0007` — Electron. Mobile local storage: `adr/0008` — a ported `expo-sqlite` storage engine. Enterprise: `adr/0009` — the multi-tenancy/RBAC/SSO model, including the circular-package-dependency fix between `@pee/auth` and `@pee/organizations`. Full detail and rationale: [03-system-architecture.md](03-system-architecture.md) and `adr/0002`-`adr/0009`.

# Frontend Checklist

Source: `SYSTEM_PROMPT.md` §107 Design Review Checklist, `claude/FRONTEND.md`.

- ☐ Consistent spacing
- ☐ Responsive layout (desktop/laptop/tablet/mobile)
- ☐ Accessible interactions (see `checklists/accessibility-checklist.md`)
- ☐ Keyboard navigation
- ☐ Theme compatibility (dark/light, dark-mode-first)
- ☐ Loading states (prefer skeletons over spinners)
- ☐ Error states (actionable, plain language)
- ☐ Empty states (educate: what/why/how to start)
- ☐ Visual hierarchy clear
- ☐ Performance targets met (see `checklists/performance-checklist.md`)
- ☐ Component reuse — no unjustified one-off components
- ☐ Design-system / token compliance
